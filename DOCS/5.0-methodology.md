# 5. Методология исследования

## 5.1 Цель эксперимента

Целью экспериментальной части данной работы является сравнение эффективности двух подходов к распознаванию эмоций по выражению лица: обучение модели на **статических данных** (одиночных кадрах) и на **динамических данных** (видеопоследовательностях).

Ключевой принцип корректности сравнения: обе модели обучаются на данных **одного и того же источника**, но в разном формате представления:

- статическая модель получает один репрезентативный кадр из каждого видео;
- динамическая модель получает последовательность кадров фиксированной длины из того же видео.

Таким образом, различие в результатах определяется прежде всего учетом временной информации, а не различием датасетов.

## 5.2 Датасет

В качестве источника данных используется набор **Urdu-Multimodal-Emotion-Dataset** (Hugging Face):

- файл метаданных `train.csv`;
- директория `video/` с видеоклипами;
- директория `audio/` (в текущем исследовании не используется).

В рамках текущего набора данных в эксперименте детектируются 5 эмоций: `anger`, `happy`, `love`, `neutral`, `sad`.

Экспериментальная часть фокусируется на **визуальной модальности** (видео лица и извлеченные кадры), без использования аудиосигнала.

Подготовка данных выполняется скриптом `prepare_data.py` и включает:

1. Чтение `train.csv` и сопоставление строк с фактически существующими видеофайлами.
2. Нормализацию меток эмоций и автоматическое построение карты классов (`label_map.json`).
3. Формирование **static-выборки**: из каждого видео извлекается один репрезентативный кадр (центральный кадр клипа).
4. Формирование **dynamic-выборки**: из каждого видео равномерно выбирается последовательность из `N=16` кадров.
5. Детектирование и кадрирование области лица (OpenCV Haar cascade) с резервным сценарием (использование полного кадра при отсутствии детекции).
6. Приведение изображений к размеру `224x224`.
7. Разбиение на train/val/test в пропорции `80/10/10` со стратификацией по классу (`splits.txt`).

Пояснение разбиения данных:

- train (обучающая выборка) используется для обновления весов модели.
- val (валидационная выборка) не участвует в обновлении весов и нужна для контроля переобучения и выбора настроек обучения (например, для early stopping).
- test (тестовая выборка) используется только для итоговой оценки качества, чтобы результаты сравнения не зависели от подгонки гиперпараметров.

Итоговые артефакты препроцессинга:

- `data/processed/static/`
- `data/processed/dynamic/`
- `data/processed/splits.txt`
- `data/processed/label_map.json`

## 5.3 Архитектуры моделей

### Статическая модель: ResNet-18

Для анализа статических данных используется архитектура **ResNet-18**. На вход подается одно изображение лица, на выходе получается распределение вероятностей по классам эмоций.

Используется transfer learning: веса ResNet-18 инициализируются предобученной моделью ImageNet, после чего классификационный слой адаптируется под число классов текущего датасета.

### Динамическая модель: ResNet-18 + LSTM

Для анализа динамических данных используется комбинированная архитектура:

1. ResNet-18 (без последнего классификационного слоя) извлекает пространственные признаки из каждого кадра последовательности.
2. Последовательность признаков передается в LSTM для моделирования временных зависимостей.
3. Выход последнего временного шага LSTM подается на классификатор эмоций.

Такое построение обеспечивает сопоставимость с static-моделью, поскольку пространственный экстрактор признаков в обеих ветках остается одинаковым.

## 5.4 Параметры обучения

| Параметр                 | Статическая модель | Динамическая модель |
| ------------------------ | ------------------ | ------------------- |
| Базовая архитектура      | ResNet-18          | ResNet-18 + LSTM    |
| Предобучение             | ImageNet           | ImageNet (ResNet)   |
| Размер входа             | 224x224            | 16 x 224x224        |
| Оптимизатор              | Adam               | Adam                |
| Начальная скорость обуч. | 0.001              | 0.001               |
| Размер батча             | 32                 | 8                   |
| Количество эпох          | 50                 | 50                  |
| Функция потерь           | CrossEntropyLoss   | CrossEntropyLoss    |
| Аугментация              | Да                 | Да                  |

Финальное число классов определяется автоматически на этапе препроцессинга на основе меток в `train.csv`.

Примечание: значения в таблице отражают базовую (baseline) конфигурацию. В рамках анти-overfitting экспериментов финальная конфигурация обучения была уточнена (optimized): learning rate снижен до `lr=1e-4`, добавлены приёмы против переобучения (например, early stopping и label smoothing). Итоговые результаты baseline vs optimized приведены в разделах 7–8.

Пояснение параметра learning rate:

- `lr` (learning rate) — скорость обучения, то есть «размер шага», с которым оптимизатор обновляет веса нейросети.
- Запись `1e-4` означает 0.0001.

Коротко о терминах обучения, которые встречаются далее:

- эпоха (epoch) — один полный проход по обучающей выборке;
- batch (батч) — порция примеров, обрабатываемая за один шаг оптимизатора;
- `weight_decay` — форма регуляризации (штраф за большие веса), помогающая снизить переобучение;
- scheduler — механизм, который автоматически меняет `lr` по ходу обучения (например, снижает его, если `val_loss` перестал улучшаться).

Ниже приведён фрагмент кода из используемого скрипта обучения, показывающий, как на практике задаются `lr`, `weight_decay` и `label_smoothing`:

```python
criterion = nn.CrossEntropyLoss(label_smoothing=args.label_smoothing)
optimizer = torch.optim.Adam(model.parameters(), lr=args.lr, weight_decay=args.weight_decay)
```

А также пример логики ранней остановки (early stopping) по отсутствию улучшения на validation:

```python
if no_improve_epochs >= args.early_stopping_patience:
	print("  Early stopping triggered.")
	break
```

Пример того, как в цикле по эпохам сравниваются метрики train и val (и как scheduler получает val_loss):

```python
train_loss, train_acc = train_one_epoch(
	model, train_loader, criterion, optimizer, DEVICE, amp_enabled, scaler
)
val_loss, val_acc = validate(model, val_loader, criterion, DEVICE, amp_enabled)
scheduler.step(val_loss)
```

## 5.5 Метрики оценки

Для сравнения моделей используются:

- **Accuracy**;
- **Precision (macro)**;
- **Recall (macro)**;
- **F1-score (macro)**;
- **confusion matrix**;
- **FPS** (скорость инференса) для оценки применимости в режиме реального времени.

Основной критерий сравнения качества между static и dynamic ветками: прирост макро-F1 при сопоставимом уровне вычислительной стоимости.

## 5.6 Инструменты и оборудование

**Программные средства:**

- Python 3.10+;
- PyTorch;
- OpenCV;
- scikit-learn;
- matplotlib.

**Аппаратная часть:**

- поддерживаются CUDA-устройства, Apple Silicon (MPS) и CPU;
- веб-камера используется для демонстрации режима реального времени.

Выбор устройства обучения выполняется автоматически (GPU при наличии, иначе CPU). Используемая логика:

```python
if torch.cuda.is_available():
    DEVICE = torch.device("cuda")
elif torch.backends.mps.is_available() and torch.backends.mps.is_built():
    DEVICE = torch.device("mps")
else:
    DEVICE = torch.device("cpu")
```
