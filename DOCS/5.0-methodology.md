# 5. Методология исследования

## 5.1 Цель эксперимента

Целью экспериментальной части данной работы является сравнение эффективности двух подходов к распознаванию эмоций по выражению лица: обучение модели на **статических данных** (отдельных изображениях) и на **динамических данных** (видеопоследовательностях). Эксперимент призван ответить на вопрос: даёт ли учёт временнóй информации при обучении модели ощутимый прирост точности по сравнению с подходом, основанным только на анализе отдельных кадров.

Для обеспечения корректности сравнения обе модели обучаются на одном и том же датасете, но используют данные в разном формате: статическая модель получает на вход пиковые кадры (моменты максимальной выраженности эмоции), а динамическая — полные последовательности кадров от нейтрального выражения до пика эмоции.

## 5.2 Датасет

В качестве источника данных используется **CK+** (_Extended Cohn-Kanade Dataset_) [32] — один из наиболее цитируемых датасетов в области распознавания эмоций по выражению лица. CK+ содержит 593 видеопоследовательности, записанные с участием 123 субъектов в контролируемых условиях (фронтальная камера, равномерное освещение).

Каждая последовательность начинается с нейтрального выражения лица и завершается пиковым выражением одной из эмоций. Из 593 последовательностей 327 имеют метки эмоций, распределённые по 7 классам: гнев (_anger_), презрение (_contempt_), отвращение (_disgust_), страх (_fear_), радость (_happiness_), грусть (_sadness_) и удивление (_surprise_).

Подготовка данных для двух моделей:

- **Статическая выборка**: из каждой размеченной последовательности извлекается **последний кадр** (пик эмоции). Таким образом формируется набор отдельных изображений с метками эмоций.
- **Динамическая выборка**: из каждой последовательности извлекаются **N последних кадров** (например, 16), представляющих переход от нейтрального состояния к пику. Короткие последовательности дополняются дублированием начальных кадров.

Для обоих вариантов выполняется:

- детектирование и обрезка области лица с помощью библиотеки OpenCV и каскадов Haar [18] или модели MTCNN [29];
- приведение изображений к единому размеру (224×224 пикселей);
- нормализация значений пикселей;
- разделение на обучающую (80%), валидационную (10%) и тестовую (10%) выборки с учётом идентичности субъектов (subject-independent split), чтобы один и тот же человек не попадал одновременно в обучающую и тестовую выборки.

## 5.3 Архитектуры моделей

### Статическая модель: ResNet-18

Для анализа статических данных используется архитектура **ResNet-18** [9] — свёрточная нейронная сеть с 18 слоями и остаточными соединениями (_residual connections_). ResNet-18 выбрана как баланс между сложностью и точностью: достаточно глубокая для извлечения информативных признаков, но компактная для быстрого обучения и инференса.

Модель инициализируется весами, предобученными на ImageNet (_transfer learning_), после чего последний полносвязный слой заменяется на новый с 7 выходами (по числу классов эмоций). Обучение проводится с дообучением всех слоёв сети.

### Динамическая модель: ResNet-18 + LSTM

Для анализа динамических данных используется комбинированная архитектура: тот же **ResNet-18** (без последнего классификационного слоя) выступает как экстрактор пространственных признаков из каждого кадра последовательности, а **LSTM-сеть** [16] обрабатывает полученную последовательность признаковых векторов, моделируя временные зависимости.

Архитектура:

1. Каждый кадр последовательности проходит через ResNet-18 → вектор признаков размерности 512.
2. Последовательность из N векторов подаётся на вход LSTM с одним или двумя скрытыми слоями.
3. Выход LSTM последнего временного шага подаётся на полносвязный слой с 7 выходами.

Использование одного и того же ResNet-18 в качестве экстрактора признаков для обеих моделей обеспечивает корректность сравнения: различие в результатах будет обусловлено именно учётом временнóй информации, а не различием в пространственных признаках.

## 5.4 Параметры обучения

| Параметр                 | Статическая модель | Динамическая модель |
| ------------------------ | ------------------ | ------------------- |
| Базовая архитектура      | ResNet-18          | ResNet-18 + LSTM    |
| Предобучение             | ImageNet           | ImageNet (ResNet)   |
| Размер входа             | 224×224 px         | 16 × 224×224 px     |
| Оптимизатор              | Adam               | Adam                |
| Начальная скорость обуч. | 0.001              | 0.001               |
| Размер батча             | 32                 | 8                   |
| Количество эпох          | 50                 | 50                  |
| Функция потерь           | CrossEntropyLoss   | CrossEntropyLoss    |
| Аугментация              | Да                 | Да                  |

Параметры могут быть скорректированы по результатам валидации.

## 5.5 Метрики оценки

Для сравнения моделей используются следующие метрики:

- **Accuracy** — доля правильно классифицированных примеров от общего числа.
- **Precision** — доля истинно положительных результатов среди всех, отнесённых к данному классу (насколько модель точна при предсказании конкретной эмоции).
- **Recall** — доля истинно положительных результатов среди всех реальных примеров данного класса (насколько полно модель находит примеры конкретной эмоции).
- **F1-score** — гармоническое среднее Precision и Recall, дающее сбалансированную оценку при неравномерном распределении классов.

Все метрики рассчитываются как **macro-average** (среднее по всем классам), а также для каждого класса эмоций отдельно. Дополнительно строится **матрица ошибок** (_confusion matrix_) для визуализации паттернов ошибок каждой модели.

Также измеряется **скорость инференса** (кадров в секунду, FPS) — для оценки пригодности моделей к работе в режиме реального времени.

## 5.6 Инструменты и оборудование

**Программные средства:**

- **Python 3.10+** — основной язык разработки.
- **PyTorch** — фреймворк глубокого обучения для построения и обучения моделей [20].
- **OpenCV** — библиотека для обработки изображений, захвата видео с камеры и детектирования лиц [19].
- **scikit-learn** — расчёт метрик (accuracy, precision, recall, F1) и построение confusion matrix.
- **matplotlib** — визуализация графиков обучения и результатов.

**Оборудование:**

- GPU: NVIDIA GeForce RTX 4080 Super (16 ГБ VRAM).
- RAM: 32 ГБ.
- Веб-камера для финальной демонстрации распознавания в реальном времени.

## 5.7 План проведения эксперимента

1. Загрузка и подготовка датасета CK+ (детектирование лиц, формирование статической и динамической выборок, разбиение на train/val/test).
2. Обучение статической модели (ResNet-18) на пиковых кадрах.
3. Обучение динамической модели (ResNet-18 + LSTM) на последовательностях кадров.
4. Оценка обеих моделей на тестовой выборке по всем метрикам.
5. Сравнительный анализ результатов (таблицы, графики, confusion matrix).
6. Интеграция лучшей модели с веб-камерой для демонстрации в реальном времени.
