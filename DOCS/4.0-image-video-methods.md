# 4. Методы анализа изображений и видео

Данный раздел посвящён двум основным типам данных, используемым для обучения моделей распознавания эмоций: статическим (отдельные изображения) и динамическим (видеопоследовательности). Каждый подход имеет свои архитектурные особенности, преимущества и ограничения.

## 4.1 Статические данные (изображения)

### Принцип обучения на статических данных

При использовании статических данных модель обучается на отдельных изображениях лиц, каждому из которых присвоена метка эмоции (например, «радость», «гнев», «нейтральное»). Основная архитектура — **свёрточная нейронная сеть** (CNN), которая принимает на вход изображение фиксированного размера и выдаёт вероятности принадлежности к каждому из классов эмоций [6].

Процесс обучения включает следующие этапы:

1. **Подготовка данных** — изображения лиц обрезаются, приводятся к единому размеру (например, 48×48 или 224×224 пикселей), нормализуются по яркости.
2. **Аугментация** — для увеличения разнообразия данных применяются случайные повороты, отражения, сдвиги, изменения яркости и контраста. Это снижает переобучение и повышает устойчивость модели [39].
3. **Обучение сети** — модель итеративно обновляет свои параметры (веса), минимизируя функцию потерь (обычно кросс-энтропию) с помощью алгоритмов оптимизации (Adam, SGD) [6].
4. **Трансферное обучение** — часто используются предобученные модели (VGG, ResNet, EfficientNet), обученные на ImageNet, с дообучением последних слоёв на датасете эмоций. Это позволяет достичь хороших результатов даже при ограниченном объёме данных [40].

На выходе обученная модель способна по одному изображению лица определить наиболее вероятную эмоцию. Для работы в реальном времени каждый кадр видеопотока обрабатывается независимо — как отдельное статическое изображение.

### Преимущества статических данных

- **Простота сбора и разметки данных.** Отдельные изображения легче получить и аннотировать, чем видеозаписи. Крупнейшие датасеты (FER2013 — 35 887 изображений [33], AffectNet — ~450 000 [34]) построены именно на статических данных.
- **Низкие вычислительные требования.** Обработка одного изображения значительно быстрее, чем анализ видеопоследовательности. Это критично для систем реального времени с ограниченными ресурсами (мобильные устройства, встраиваемые системы).
- **Зрелость подходов.** CNN-архитектуры для классификации изображений хорошо изучены, доступно большое количество предобученных моделей и библиотек [40].

### Ограничения статических данных

- **Потеря временнóй информации.** Статическое изображение фиксирует лишь один момент — пик эмоции. Переходные состояния, последовательность мимических движений и динамика изменения выражения лица не учитываются [41].
- **Неоднозначность отдельных кадров.** На отдельном изображении начало удивления может быть неотличимо от начала страха. Контекст предыдущих и последующих кадров отсутствует.
- **Чувствительность к качеству снимка.** Размытие, неудачный ракурс или частичное перекрытие лица на единственном кадре приводят к ошибке, тогда как в видео такие кадры могут быть компенсированы соседними.

## 4.2 Динамические данные (видео)

### Принцип обучения на динамических данных

При использовании динамических данных модель обучается на **последовательностях кадров** (видеоклипах), что позволяет учитывать изменение выражения лица во времени. Для этого используются архитектуры, способные моделировать временные зависимости [41]:

- **CNN + RNN/LSTM** — свёрточная сеть извлекает пространственные признаки из каждого кадра, а рекуррентная сеть (LSTM или GRU) обрабатывает последовательность этих признаков во времени, выявляя динамические паттерны [16].
- **3D-CNN** — трёхмерные свёрточные сети применяют фильтры не только по пространственным осям (высота, ширина), но и по временнóй оси, обрабатывая блоки из нескольких последовательных кадров одновременно. Архитектура C3D является одной из наиболее известных реализаций данного подхода [42].
- **Трансформеры с временным вниманием** — современные подходы используют механизм внимания (_attention_) для определения наиболее информативных кадров в последовательности, что позволяет гибко взвешивать вклад каждого момента времени [43].

Процесс обучения аналогичен статическому подходу (подготовка, аугментация, оптимизация), но дополнительно включает выбор длины входной последовательности (например, 16 или 32 кадра), стратегию выборки кадров (_sampling_) и синхронизацию пространственных и временных признаков.

### Преимущества динамических данных

- **Учёт временнóй динамики.** Эмоции — это процесс, а не мгновенное состояние. Видеоданные позволяют наблюдать начало, кульминацию и затухание эмоционального выражения, что предоставляет модели существенно больше информации [41].
- **Распознавание микровыражений.** Микровыражения — кратковременные (длительностью 1/25–1/5 секунды) непроизвольные мимические движения, которые невозможно уловить на отдельном изображении. Анализ видео позволяет их детектировать [44].
- **Устойчивость к шуму.** Если отдельный кадр повреждён или неинформативен (размытие, моргание), модель может опираться на информацию из соседних кадров последовательности.

### Ограничения динамических данных

- **Высокие вычислительные затраты.** Обработка видеопоследовательностей требует значительно больше памяти и вычислительных мощностей. Архитектуры CNN+LSTM и 3D-CNN обрабатывают данные медленнее, чем статические CNN, что может быть критично для приложений реального времени [42].
- **Ограниченность датасетов.** Видеодатасеты эмоций существенно меньше статических: RAVDESS содержит 7 356 записей [35], CK+ — 593 последовательности [32], тогда как AffectNet — ~450 000 изображений. Сбор и разметка видеоданных значительно трудоёмче.
- **Сложность разметки.** Определение точного момента начала и завершения эмоции в видео — нетривиальная задача, подверженная субъективности аннотаторов.

## 4.3 Сравнительный обзор

| Критерий                   | Статические данные | Динамические данные            |
| -------------------------- | ------------------ | ------------------------------ |
| Входные данные             | Одно изображение   | Последовательность кадров      |
| Архитектуры                | CNN (VGG, ResNet)  | CNN+LSTM, 3D-CNN, трансформеры |
| Временная информация       | Не учитывается     | Моделируется                   |
| Вычислительные требования  | Низкие             | Высокие                        |
| Размер доступных датасетов | Большие (>100 000) | Небольшие (<10 000)            |
| Скорость инференса         | Высокая            | Средняя / низкая               |
| Микровыражения             | Не распознаются    | Могут распознаваться           |

Ряд исследований показывает, что динамические модели превосходят статические по точности при наличии достаточного объёма данных. Например, в обзоре Li и Deng (2022) отмечается, что комбинированные архитектуры CNN+LSTM демонстрируют прирост точности на 2–5% по сравнению с чисто пространственными CNN на видеодатасетах [30]. Однако при ограниченных вычислительных ресурсах и в задачах реального времени статические подходы остаются конкурентоспособными благодаря своей скорости и простоте.

Данное исследование направлено на экспериментальную проверку этих различий: в практической части работы будут обучены и сравнены модели обоих типов на одних и тех же данных.
