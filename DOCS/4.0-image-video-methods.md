# 4. Методы анализа изображений и видео

Данный раздел посвящён двум основным типам данных, используемым для обучения моделей распознавания эмоций: статическим (отдельные изображения) и динамическим (видеопоследовательности). Каждый подход имеет свои архитектурные особенности, преимущества и ограничения.

## 4.1 Статические данные (изображения)

### Принцип обучения на статических данных

При использовании статических данных модель обучается на отдельных изображениях лиц, каждому из которых присвоена метка эмоции (например, «радость», «гнев», «нейтральное»). Основная архитектура — **свёрточная нейронная сеть** (CNN), которая принимает на вход изображение фиксированного размера и выдаёт вероятности принадлежности к каждому из классов эмоций [4].

Процесс обучения включает следующие этапы:

1. **Подготовка данных** — изображения лиц обрезаются, приводятся к единому размеру (например, 48×48 или 224×224 пикселей), нормализуются по яркости.
2. **Аугментация** — для увеличения разнообразия данных применяются случайные повороты, отражения, сдвиги, изменения яркости и контраста. Это снижает переобучение и повышает устойчивость модели [17].
3. **Обучение сети** — модель итеративно обновляет свои параметры (веса), минимизируя функцию потерь (обычно кросс-энтропию) с помощью алгоритмов оптимизации (Adam, SGD) [4].
4. **Трансферное обучение** — часто используются предобученные модели (например, ResNet), обученные на ImageNet, с дообучением последних слоёв на датасете эмоций. Это позволяет достичь хороших результатов даже при ограниченном объёме данных [7].

На выходе обученная модель способна по одному изображению лица определить наиболее вероятную эмоцию. Для работы в реальном времени каждый кадр видеопотока обрабатывается независимо — как отдельное статическое изображение.

### Преимущества статических данных

- **Простота сбора и разметки данных.** Отдельные изображения легче получить и аннотировать, чем видеозаписи. Многие крупные датасеты построены именно на статических данных; пример — AffectNet [16].
- **Низкие вычислительные требования.** Обработка одного изображения значительно быстрее, чем анализ видеопоследовательности. Это критично для систем реального времени с ограниченными ресурсами (мобильные устройства, встраиваемые системы).
- **Зрелость подходов.** CNN-архитектуры для классификации изображений хорошо изучены, доступно большое количество предобученных моделей и библиотек [4].

### Ограничения статических данных

- **Потеря временнóй информации.** Статическое изображение фиксирует лишь один момент. Переходные состояния и динамика изменения выражения лица не учитываются.
- **Неоднозначность отдельных кадров.** На отдельном изображении начало удивления может быть неотличимо от начала страха. Контекст предыдущих и последующих кадров отсутствует.
- **Чувствительность к качеству снимка.** Размытие, неудачный ракурс или частичное перекрытие лица на единственном кадре приводят к ошибке, тогда как в видео такие кадры могут быть компенсированы соседними.

## 4.2 Динамические данные (видео)

### Принцип обучения на динамических данных

При использовании динамических данных модель обучается на **последовательностях кадров** (видеоклипах), что позволяет учитывать изменение выражения лица во времени. На практике распространён подход **CNN + LSTM**: свёрточная сеть извлекает пространственные признаки из каждого кадра, а LSTM обрабатывает последовательность этих признаков во времени, выявляя динамические паттерны [8].

Процесс обучения аналогичен статическому подходу (подготовка, аугментация, оптимизация), но дополнительно включает выбор длины входной последовательности (например, 16 или 32 кадра), стратегию выборки кадров (_sampling_) и синхронизацию пространственных и временных признаков.

### Преимущества динамических данных

- **Учёт временнóй динамики.** Эмоции — это процесс, а не мгновенное состояние. Видеоданные позволяют наблюдать изменения выражения лица во времени, что может дать модели больше информации.
- **Устойчивость к шуму.** Если отдельный кадр повреждён или неинформативен (размытие, моргание), модель может опираться на информацию из соседних кадров последовательности.

### Ограничения динамических данных

- **Высокие вычислительные затраты.** Обработка видеопоследовательностей требует значительно больше памяти и вычислительных мощностей, поэтому скорость (FPS) обычно ниже, чем у статического подхода.
- **Ограниченность датасетов.** Видеоданные труднее собирать и размечать, поэтому видеодатасеты эмоций часто меньше и менее разнообразны, чем датасеты статических изображений.
- **Сложность разметки.** Определение точного момента начала и завершения эмоции в видео — нетривиальная задача, подверженная субъективности аннотаторов.

## 4.3 Сравнительный обзор

| Критерий                   | Статические данные | Динамические данные            |
| -------------------------- | ------------------ | ------------------------------ |
| Входные данные             | Одно изображение   | Последовательность кадров      |
| Архитектуры                | CNN (например, ResNet)  | CNN+LSTM |
| Временная информация       | Не учитывается     | Моделируется                   |
| Вычислительные требования  | Низкие             | Высокие                        |
| Размер доступных датасетов | Большие (>100 000) | Небольшие (<10 000)            |
| Скорость инференса         | Высокая            | Средняя / низкая               |
| Микровыражения             | Не распознаются    | Могут распознаваться           |

Ряд исследований показывает, что динамические модели превосходят статические по точности при наличии достаточного объёма данных. Например, в обзоре Li и Deng (2022) отмечается, что комбинированные архитектуры CNN+LSTM демонстрируют прирост точности на 2–5% по сравнению с чисто пространственными CNN на видеодатасетах [15]. Однако при ограниченных вычислительных ресурсах и в задачах реального времени статические подходы остаются конкурентоспособными благодаря своей скорости и простоте.

В практической части данной работы оба подхода обучаются на одном источнике видео (Urdu-Multimodal-Emotion-Dataset): для static-ветки используется один репрезентативный кадр из каждого клипа, а для dynamic-ветки — последовательность из 16 кадров. Такой дизайн минимизирует искажения сравнения, связанные с различием датасетов.
