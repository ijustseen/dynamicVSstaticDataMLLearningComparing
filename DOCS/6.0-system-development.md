# 6. Разработка системы распознавания эмоций

После описания используемых данных и методологии эксперимента перейдём к практической реализации системы распознавания эмоций в режиме реального времени. Важная цель разработки — не только добиться высокой точности на обучающей выборке, но и обеспечить устойчивое качество на новых данных.

## 6.1 Архитектура системы

Система реализована в виде последовательного конвейера обработки видеопотока:

1. Захват кадров с веб-камеры.
2. Детектирование области лица на кадре.
3. Предобработка изображения лица (приведение к размеру входа модели, нормализация).
4. Передача данных в модель классификации эмоций.
5. Получение вероятностей классов и выбор итоговой эмоции.
6. Визуализация результата поверх видеопотока.

Логическая схема конвейера:

Камера -> Детектирование лица -> Предобработка -> Модель -> Класс эмоции -> Отображение результата

Для сравнительного анализа использовались два режима обработки:

- статический режим (обработка отдельных изображений);
- динамический режим (обработка последовательностей кадров с учетом временного контекста).

## 6.2 Используемые технологии

В работе использован следующий программный стек:

- Python как основной язык разработки;
- OpenCV для работы с камерой и первичной обработки изображений;
- PyTorch (с поддержкой CUDA при наличии GPU) для построения и обучения моделей;
- NumPy и Pandas для обработки данных и анализа результатов;
- Matplotlib для визуализации кривых обучения.

Выбор данного стека обусловлен его пригодностью для быстрых итераций, воспроизводимости экспериментов и единого подхода к работе как со статическими, так и с динамическими данными.

## 6.3 Подготовка данных

Подготовка данных включала следующие этапы:

1. Формирование датасета по классам эмоций.
2. Очистка от некачественных и нерелевантных примеров.
3. Приведение изображений к единому формату.
4. Нормализация входных признаков.
5. Разделение на обучающую, валидационную и тестовую выборки.

Для повышения устойчивости моделей применялась аугментация данных:

- горизонтальное отражение;
- небольшие повороты;
- вариации яркости и контраста;
- легкие геометрические преобразования.

Данные меры направлены на снижение чувствительности модели к частным особенностям обучающей выборки.

## 6.4 Обучение моделей и выявленная сложность

В ходе первых серий запусков были зафиксированы признаки переобучения:

- устойчивый рост метрик на обучающей выборке;
- остановка роста или снижение метрик на валидационной выборке;
- увеличение разрыва между train и validation.

Указанная картина свидетельствовала о том, что модель запоминает особенности обучающих примеров и в меньшей степени формирует обобщающие признаки.

Проще говоря, переобучение — это ситуация, когда модель «очень хорошо» работает на тех данных, на которых её обучали (train), но заметно хуже — на новых данных (validation/test). Поэтому в работе отдельно отслеживались кривые обучения и разница между train и validation (generalization gap).

Для системного управления данной сложностью был введен отдельный цикл анти-overfitting экспериментов:

1. Фиксация базового запуска (baseline).
2. Анализ кривых обучения и разрыва train/validation.
3. Изменение одного фактора за запуск.
4. Повторная оценка метрик.
5. Документирование результата и вывода.

В качестве корректирующих мер рассматривались:

- ранняя остановка обучения (early stopping);
- регуляризация (dropout, weight decay);
- усиление аугментации данных;
- настройка learning rate и scheduler;
- снижение избыточной сложности модели.

На практике применялась ограниченная серия контролируемых анти-overfitting итераций (с фиксацией baseline, оценкой на test и документированием выводов), что позволило выбрать финальную конфигурацию обучения без длительного подбора гиперпараметров.

Таким образом, переобучение зафиксировано как значимая инженерная и исследовательская сложность, для которой применен формализованный подход к снижению.

## 6.5 Интеграция с камерой

После стабилизации обучения выполнена интеграция модели с видеопотоком камеры в режиме реального времени.

Основные особенности реализации:

- непрерывное чтение кадров;
- инференс по каждому кадру или по каждому N-му кадру для баланса качества и производительности;
- визуализация предсказания на кадре;
- контроль частоты кадров (FPS) как практического показателя применимости.

Для снижения флуктуаций предсказания может использоваться сглаживание по окну последних кадров (temporal smoothing).

## 6.6 Выводы по разделу

В результате разработки сформирована работоспособная система распознавания эмоций, объединяющая этапы подготовки данных, обучения и инференса в реальном времени. Ключевым практическим результатом этапа стало не только построение рабочего конвейера, но и выявление переобучения с переходом к структурированному циклу его снижения.
