# 7. Проведение эксперимента

После разработки системы и подготовки конвейера обучения приведём ход экспериментальной части: сравнение моделей на статических и динамических данных, а также диагностика и интерпретация переобучения.

## 7.1 Цель и организация эксперимента

Цели эксперимента:

- сравнить качество распознавания эмоций для статического и динамического подходов;
- оценить устойчивость моделей на валидационной и тестовой выборках;
- зафиксировать признаки переобучения и определить направления его снижения.

Экспериментальная работа выполнялась в формате воспроизводимых запусков с единым протоколом оценки.

Пояснение терминов (в простом виде):

- статический подход — модель получает один кадр (изображение) и предсказывает эмоцию;
- динамический подход — модель получает последовательность кадров и использует временной контекст;
- baseline — базовый вариант обучения (например, с `lr=1e-3`), который нужен как точка сравнения;
- optimized — улучшенный вариант обучения (в финальной конфигурации `lr=1e-4` + приёмы против переобучения), который сравнивается с baseline;
- anti-overfitting — серия контролируемых запусков, где за один раз меняется один фактор (например, early stopping или `lr`), чтобы уменьшить переобучение и улучшить качество на новых данных.

## 7.2 Протокол оценки

Для каждого запуска фиксировались следующие показатели:

- train_loss и val_loss;
- train_acc и val_acc;
- итоговые test-метрики (accuracy, precision, recall, F1-score);
- скорость инференса (FPS) при работе с камерой.

Пояснение разбиения данных:

- train (обучающая выборка) — данные, на которых обновляются веса нейросети;
- val (валидационная выборка) — отдельная часть данных для контроля переобучения и выбора настроек обучения (например, для early stopping);
- test (тестовая выборка) — данные, которые не используются при настройке модели и применяются для итоговой проверки качества.

В данной работе используется разбиение `80/10/10` (train/val/test) со стратификацией по классу.

Примечание по метрикам:

- Accuracy показывает долю правильных ответов.
- Precision/Recall/F1-score показывают качество по классам эмоций; в таблицах используются усреднённые значения (macro average), чтобы редкие эмоции не «терялись» на фоне частых.
- FPS показывает практическую применимость в реальном времени: чем выше FPS, тем плавнее работает система.

Для количественной диагностики переобучения использовался показатель generalization gap:

gap = train_accuracy - val_accuracy

## 7.5 Переобучение как исследовательская сложность

Выполненные запуски подтвердили, что рост метрик на обучающей выборке не является достаточным критерием качества модели. В исследовании данное явление зафиксировано как отдельная инженерная сложность, требующая системного подхода.

Принятый рабочий цикл:

1. Фиксация baseline-результатов.
2. Анализ кривых обучения и значения gap.
3. Внесение одного изменения за запуск.
4. Повторная оценка и сравнение с baseline.
5. Документирование полученного вывода.

Такой подход обеспечивает прозрачность экспериментальных решений и уменьшает риск неконтролируемых изменений.

## 7.6 Итоги серии анти-overfitting экспериментов

В рамках дипломной работы выполнена ограниченная серия контролируемых экспериментов (без длительного перебора гиперпараметров), в результате чего выбрана итоговая конфигурация:

- anti-overfitting меры: early stopping, регуляризация и сглаживание меток (label smoothing);
- критерий выбора чекпоинта для итоговой модели: best-by-acc (по максимуму val_acc);
- подобранный learning rate: lr=1e-4 (0.0001).

Для контроля устойчивости была выполнена короткая проверка альтернативного learning rate (lr=3e-4), которая показала ухудшение качества (особенно для динамического режима), поэтому финальной конфигурацией оставлен вариант с lr=1e-4.

Пояснение для читателя: learning rate (скорость обучения) определяет «размер шага» оптимизатора. Слишком большой шаг может приводить к нестабильному обучению и ухудшению качества на новых данных, что и наблюдалось при проверке lr=3e-4.

## 7.7 Выводы по разделу

Экспериментальная часть показала преимущество динамического подхода по качеству распознавания эмоций и одновременно выявила компромисс между точностью и скоростью работы. В обоих подходах обнаружены признаки переобучения, что обусловило переход к формализованному циклу контролируемых экспериментов с фокусом на обобщающую способность.

Тем самым в рамках исследования получены два ключевых практических результата: количественное сравнение подходов и методически обоснованная стратегия работы с переобучением.

Далее эти результаты интерпретируются с практической точки зрения (компромисс качество/FPS, характер ошибок и причины выбора финальной конфигурации).
